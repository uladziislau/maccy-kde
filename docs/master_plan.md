# Манифест реализации maccy-kde (Бриллиантовый Стек)

## 1. Цель
Создать идеальный визуальный и функциональный клон Maccy для KDE Plasma 6 (Wayland) с нулевым потреблением ресурсов в простое.

## 2. Максимально Эффективный Стек
- **База данных:** `rusqlite` (легче и быстрее, чем sqlx, < 7 МБ RAM).
- **Слушатель буфера:** `smithay-clipboard` в фоновом потоке. Полностью Event-Driven (никакого поллинга!). Процесс спит, пока Wayland не пришлет событие.
- **Глобальные Хоткеи:** `zbus` (интеграция с `org.kde.kglobalaccel`). Приложение - это единый фоновый процесс. Мы не плодим процессы, а просто показываем/скрываем окно Slint по сигналу от KWin.
- **Интерфейс:** `Slint` (нативный Wayland, мгновенный рендер).
- **Вставка текста:** `enigo` (или `zbus` KWin API) для симуляции `Ctrl+V`.

## 3. Архитектура данных
Таблица `clipboard_items`:
- `id` (PRIMARY KEY)
- `value_text` (TEXT)
- `is_pinned` (INTEGER DEFAULT 0)
- `pin_order` (INTEGER DEFAULT 0)
- `last_used_at` (INTEGER)

## 4. Поведение
- При старте `maccy-kde` уходит в фон, регистрирует глобальный хоткей через `zbus` и начинает слушать буфер через `smithay-clipboard`.
- При нажатии хоткея мгновенно всплывает окно `Slint` под курсором мыши.
- Пользователь выбирает элемент (Enter, `Alt+1..9`).
- Окно скрывается, текст устанавливается в системный буфер, эмулируется `Ctrl+V`.
