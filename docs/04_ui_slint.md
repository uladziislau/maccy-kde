# 04. Графический интерфейс на Slint UI и сборка Flatpak

Этот документ описывает правила построения пользовательского интерфейса в стиле Maccy и упаковки приложения.

## 1. Дизайн и поведение GUI (`maccy-kde --popup`)
Интерфейс должен быть полностью Keyboard-First (мышь опциональна) и использовать фреймворк **Slint**.

- **Появление:** Окно должно создаваться без системных рамок (`no-frame: true`) прямо в координатах текущего положения курсора мыши.
- **Поиск на лету:** Фокус ввода всегда находится в поле `LineEdit`. При вводе любого символа массив истории фильтруется на лету с использованием крейта `fuzzy-matcher`.
- **Быстрые клавиши:** 
  - Нажатие `Enter` — вставляет первый (верхний) элемент из списка.
  - Нажатие `Alt+1` .. `Alt+9` — мгновенно выбирает и вставляет элемент под соответствующим номером.
  - Нажатие `Esc` — закрывает приложение без вставки.

## 2. Нативная интеграция с KDE KWin (Эффект размытия)
Чтобы окно имело полупрозрачный размытый фон в стиле macOS, в коде инициализации Slint-окна необходимо через `zbus` отправить окну свойство для KWin:
- Установить атомарное свойство оконного менеджера `_KDE_NET_WM_BLUR_BEHIND_REGION`.

## 3. Манифест Flatpak (Песочница)
Поскольку приложение работает в изоляции Flatpak, ИИ должен сгенерировать файл манифеста `org.maccy_kde.ClipboardManager.yml`. В манифесте обязательно должны быть открыты следующие шлюзы:

```yaml
id: org.maccy_kde.ClipboardManager
runtime: org.kde.Platform
runtime-version: '6.6' # Окружение KDE Plasma 6
sdk: org.kde.Sdk
command: maccy-kde
finish-args:
  - --socket=wayland           # Доступ к дисплею и буферу обмена
  - --socket=fallback-x11      # На всякий случай для обратной совместимости
  - --socket=session-bus       # Доступ к DBus для эмуляции клавиатуры KWin и хоткеев
  - --filesystem=xdg-data/maccy-kde:create # Доступ к локальной БД истории
  - --filesystem=xdg-cache/maccy-kde:create # Доступ к кэшу изображений
```
